Документация API

Аутентификация

POST /register
Тело: 
{
  "email": "user@example.com",
  "password": "password",
  "name": "Имя"
}
Ответ: 
{
  "success": true,
  "token": "JWT_ТОКЕН",
  "user": { ... }
}
POST /login
Тело: 
{
  "email": "user@example.com",
  "password": "password"
}
Ответ: 
{
  "success": true,
  "token": "JWT_ТОКЕН",
  "user": { ... }
}

Для всех остальных запросов требуется заголовок:
Authorization: Bearer JWT_ТОКЕН

Клубы

GET /clubs
Ответ: список клубов.
Пример: 
{
  "clubs": [
    {
      "id": 1,
      "name": "Night Club X",
      "description": "Лучший клуб города!",
      "latitude": 55.751244,
      "longitude": 37.618423,
      "rating": 4.5,
      "created_at": "2024-06-07 12:00:00"
    },
    {
      "id": 2,
      "name": "Bar Y",
      "description": "Уютный бар для своих.",
      "latitude": 55.760186,
      "longitude": 37.618711,
      "rating": 4.0,
      "created_at": "2024-06-07 12:00:00"
    }
  ]
}
Поля:
  • id — идентификатор клуба
  • name — название клуба
  • description — описание
  • latitude, longitude — координаты
  • rating — средний рейтинг
  • created_at — дата создания
GET /clubs/{id}
Ответ: подробная информация о клубе
POST /clubs/{id}/reviews
Тело: 
{
  "rating": 5,
  "review": "Отличное место!"
}
Ответ: { "success": true }

Мероприятия

GET /events
Параметры: type_id, date, lat, lng, radius_km, city (опционально)
Ответ: список мероприятий.
Пример: 
{
  "events": [
    {
      "id": 1,
      "club_id": 1,
      "creator_id": 1,
      "title": "Большая вечеринка",
      "description": "Танцы до утра!",
      "latitude": 55.751244,
      "longitude": 37.618423,
      "city": "Москва",
      "start_time": "2024-06-07 21:00:00",
      "end_time": "2024-06-07 23:59:59",
      "type_id": 1,
      "status": "approved",
      "created_at": "2024-06-07 12:00:00"
    }
  ]
}
Поля:
  • id — идентификатор мероприятия
  • club_id — идентификатор клуба (если есть)
  • creator_id — кто создал
  • title — название
  • description — описание
  • latitude, longitude — координаты
  • city — город
  • start_time, end_time — время проведения
  • type_id — тип мероприятия
  • status — статус (approved, pending, rejected)
  • created_at — дата создания
GET /events/{id}
Ответ: подробная информация о мероприятии.
POST /events/{id}/attend
Ответ: { "success": true }
DELETE /events/{id}/attend
Ответ: { "success": true }
POST /events/{id}/comments
Тело: 
{
  "comment": "Очень жду!"
}
Ответ: { "success": true }

Избранное

POST /clubs/{id}/favorite
Ответ: { "success": true }
DELETE /clubs/{id}/favorite
Ответ: { "success": true }
GET /me/favorites
Ответ: список избранных клубов пользователя.
Пример: 
{
  "clubs": [
    { "id": 1, "name": "Night Club X", ... }
  ]
}

Мои мероприятия

GET /me/attending
Ответ: список мероприятий, на которые записан пользователь.
Пример: 
{
  "events": [
    { "id": 1, "title": "Большая вечеринка", ... }
  ]
}

Ошибки

{ "error": "Текст ошибки" }

Если не передан токен или он невалиден — будет ошибка 401.
