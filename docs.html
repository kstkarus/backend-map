Документация API

Аутентификация

POST /register
Тело: 
{
  "email": "user@example.com",
  "password": "password",
  "name": "Имя"
}
Ответ: 
{
  "success": true,
  "token": "JWT_ТОКЕН",
  "user": { ... }
}
POST /login
Тело: 
{
  "email": "user@example.com",
  "password": "password"
}
Ответ: 
{
  "success": true,
  "token": "JWT_ТОКЕН",
  "user": { ... }
}

Для всех остальных запросов требуется заголовок:
Authorization: Bearer JWT_ТОКЕН

Клубы

GET /clubs
Ответ: список клубов
GET /clubs/{id}
Ответ: подробная информация о клубе
POST /clubs/{id}/reviews
Тело: 
{
  "rating": 5,
  "review": "Отличное место!"
}
Ответ: { "success": true }

Мероприятия

GET /events
Параметры: type_id, date, lat, lng, radius_km, city (опционально)
Ответ: список мероприятий.
GET /events/{id}
Ответ: подробная информация о мероприятии.
POST /events/{id}/attend
Ответ: { "success": true }
DELETE /events/{id}/attend
Ответ: { "success": true }
POST /events/{id}/comments
Тело: 
{
  "comment": "Очень жду!"
}
Ответ: { "success": true }

Ошибки

{ "error": "Текст ошибки" }

Если не передан токен или он невалиден — будет ошибка 401.
